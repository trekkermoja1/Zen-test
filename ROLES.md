# Project Roles and Responsibilities

This document defines the key roles in the Zen-AI-Pentest project and their responsibilities.

## Project Maintainer

**Current:** ATAKAN_AYDIN (GitHub: [@SHAdd0WTAka](https://github.com/SHAdd0WTAka))

### Responsibilities
- Set overall project direction and vision
- Make final decisions on disputes
- Manage releases and versioning
- Oversee security policies
- Ensure project sustainability
- Approve significant architectural changes
- Maintain project infrastructure (GitHub, domains, etc.)

### Authority
- Write access to all repositories
- Ability to merge pull requests
- Administrative access to project settings

## Technical Lead

**Current:** ATAKAN_AYDIN (acting, seeking additional contributors)

### Responsibilities
- Define technical architecture
- Set code quality standards
- Review complex pull requests
- Maintain technical roadmap
- Mentor contributors
- Make technology stack decisions

### Authority
- Merge technical pull requests
- Request changes to code submissions
- Approve architectural changes

## Security Team Lead

**Current:** ATAKAN_AYDIN (acting)

### Responsibilities
- Respond to security vulnerability reports
- Coordinate security fixes
- Maintain security policies
- Review security-related code changes
- Manage responsible disclosure process

### Authority
- Access to security issues
- Ability to create security advisories
- Emergency release authority for security fixes

## Core Contributors

**Current:** Seeking active contributors

### Responsibilities
- Regular code contributions
- Code review for pull requests
- Issue triage and support
- Documentation maintenance
- Community engagement

### Authority
- Write access to repository (after demonstrating consistent contributions)
- Ability to merge approved pull requests
- Triage issues and label them

## Contributors

**Open to all**

### Responsibilities
- Submit bug reports and feature requests
- Submit code via pull requests
- Improve documentation
- Help other community members
- Follow the Code of Conduct

### Requirements
- No special requirements
- All contributions welcome
- Must sign DCO for code contributions

## Documentation Lead

**Current:** Open position

### Responsibilities
- Maintain project documentation
- Ensure docs are up-to-date with code
- Improve documentation clarity
- Manage translation efforts

## Community Manager

**Current:** Open position

### Responsibilities
- Facilitate community discussions
- Organize community events
- Onboard new contributors
- Moderate community spaces

## How to Join

### Becoming a Contributor
1. Start contributing (code, docs, issues)
2. Follow the Code of Conduct
3. Sign off commits per DCO

### Becoming a Core Contributor
1. Consistent contributions over time
2. Demonstrated understanding of project
3. Nominated by existing core contributors
4. Approved by Project Maintainer
