# Project Governance

This document describes the governance model for the Zen-AI-Pentest project.

## Overview

Zen-AI-Pentest follows an open governance model that encourages community
participation and transparent decision-making.

## Decision Making

### Consensus-Based Approach

We strive for consensus on all major decisions. When consensus cannot be reached,
the project maintainer makes the final decision after considering all input.

### Types of Decisions

| Type | Description | Decision Maker |
|------|-------------|----------------|
| **Technical** | Code changes, architecture decisions | Technical Lead + Community Input |
| **Release** | Version releases, roadmap changes | Project Maintainer |
| **Policy** | Licensing, governance, CoC changes | Project Maintainer |
| **Security** | Security policies, vulnerability handling | Security Team Lead |

### Technical Lead

- **Responsibilities:**
  - Architecture decisions
  - Code review standards
  - Technical roadmap
  - Mentoring contributors

### Core Contributors

- Regular contributors with write access
- Responsibilities:
  - Code review
  - Issue triage
  - Documentation
  - Community support

### Contributors

- Anyone who contributes to the project (code, docs, issues, etc.)
- No special permissions required to contribute

## Communication Channels

- **GitHub Issues:** Bug reports, feature requests
- **GitHub Discussions:** General questions, ideas
- **Pull Requests:** Code contributions
- **Security Issues:** security@zen-ai-pentest.dev (private)

## Conflict Resolution

1. **Discussion:** Parties discuss the issue openly
2. **Mediation:** Technical Lead facilitates resolution
3. **Decision:** Project Maintainer makes final decision if needed

## Changes to Governance

Changes to this governance model require:
1. Public discussion via GitHub issue
2. At least one week for community feedback
3. Approval by Project Maintainer

## Transparency

All project operations are conducted transparently:
- Public issue tracking
- Public code review
- Public roadmap
- Public meeting notes (when applicable)
